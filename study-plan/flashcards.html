Which pattern? — Two Pointers vs Sliding Window vs Prefix Sum, and signal words

All three walk an array with indices, so problems look alike. Pick by what the answer is and whether the constraint is monotonic.

Two Pointers — answer is a pair / tuple

  • Input is sorted (or you can sort without breaking the answer).
  • Comparing the two end values proves which pointer is safe to move — one comparison discards all pairs on one side.
  • Pointers move toward each other; you're picking elements, not a range.
Tells: "sorted", "find a pair/triplet", "two ends", palindromes, in-place partition.

Sliding Window — answer is the best contiguous run

  • "Longest / shortest / max subarray or substring such that…"
  • Constraint is monotonic: growing only violates more, shrinking only fixes (all-positive sums, distinct-char counts, "at most K").
  • Both pointers move the same direction; a running state is updated incrementally.
Tells: "longest substring without…", "minimum window containing…", "at most K distinct".

Prefix Sum — answer counts / sums ranges exactly

  • "How many subarrays sum to exactly K", or many repeated range-sum queries.
  • Works where a window breaks: negatives or exact targets, since sum(i,j)=prefix[j]−prefix[i] needs no monotonicity.
  • Pair with a hash map of seen prefixes → find current−K in O(1).
Tells: "sum equals K", "count the subarrays", "equal 0s and 1s", static array + many queries.
10-second litmus test: "If I extend the window by one element, does the constraint always move one direction?" Yes (all-positive sum grows, distinct count never drops) → sliding window. No (negatives, or target is == K not ≤ K) → prefix sum + hash map. If it's about which elements rather than which range and sorted order helps → two pointers. Examples: "longest subarray, sum ≤ K, all positive" = window; "count subarrays, sum == K, negatives" = prefix sum; "two numbers add to K, sorted" = two pointers.
