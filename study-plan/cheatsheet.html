JavaScript / TypeScript — Interview Cheat Sheet

Know these cold so you spend brainpower on the algorithm, not the syntax.

Gotchas

  • Default sort is lexicographic. [10, 2, 1].sort() → [1, 10, 2]. Always pass a comparator: arr.sort((a, b) => a - b).
  • No built-in heap/priority queue. Write your own (below). This is the #1 JS interview gap.
  • Map preserves insertion order and allows any key type — prefer over {} for frequency counts when keys aren't strings.
  • Set for O(1) membership: new Set(arr), .has(x), .add(x).
  • Integer division: Math.floor(a / b). Beware negatives: Math.trunc for truncation toward zero.
  • Number.MAX_SAFE_INTEGER ≈ 9e15. Use Infinity / -Infinity for init in min/max.
  • Copying: [...arr] (shallow). 2D grid copy: grid.map(r => [...r]).
  • Char ↔ code: 'a'.charCodeAt(0) → 97; String.fromCharCode(97) → 'a'.

Essential methods to know cold

Array — the workhorses (all return a NEW array / value)

arr.forEach((val, i) => { ... })        // side effects only, no return
arr.map((val, i) => val * 2)            // NEW array, same length
arr.filter((val) => val > 0)            // NEW array, subset
arr.reduce((acc, val) => acc + val, 0)  // fold to one value; 0 = initial

arr.includes(3)         // true/false — "does it contain"
arr.indexOf(3)          // index or -1
arr.find(v => v > 5)    // first VALUE matching, or undefined
arr.findIndex(v => v>5) // first INDEX matching, or -1
arr.some(v => v > 5)    // true if ANY match
arr.every(v => v > 0)   // true if ALL match

Array — mutation (changes the array in place)

arr.push(x)        // add to END,   returns new length   → STACK with pop()
arr.pop()          // remove END,   returns removed item
arr.shift()        // remove FRONT, returns item — O(n), avoid in loops
arr.unshift(x)     // add to FRONT, returns new length — O(n)
arr.reverse()      // in place
arr.splice(i, n)   // remove n items at i; can also insert
arr.fill(0)        // fill all slots; new Array(n).fill(0) is the idiom

slice vs splice: arr.slice(1, 3) returns a COPY of [1,3) and leaves the original alone; arr.splice(1, 3) CUTS items out and mutates. Mnemonic: splice spoils the original.

String — every two-pointer / parsing problem

str.length              // property, no parens
str[i] / str.charAt(i)  // single char
str.charCodeAt(i)       // ASCII/Unicode code — anagram & counting tricks
str.split("")           // string → array of chars (then map/sort/etc.)
str.split(" ")          // split on delimiter
arr.join("")            // array → string back
str.slice(1, 3)         // substring; negatives ok: slice(-1) = last char
str.substring(1, 3)     // like slice but no negatives
str.toLowerCase()       // valid-palindrome staple
str.trim()              // strip whitespace ends
str.replace(/[^a-z0-9]/gi, "")  // strip non-alphanumeric
str.repeat(3) / str.padStart(3, "0")  // "5" → "005"

Strings are immutable — every "modify" returns a new string. To mutate: split("") → work on the array → join("").

Building arrays / ranges

Array.from({length: n}, (_, i) => i)   // [0, 1, ..., n-1]
Array.from({length: m}, () => new Array(n).fill(0))  // m×n grid of 0s
[...arr]                  // shallow copy
grid.map(r => [...r])     // deep-ish copy of a 2D grid

Frequency count — the most common pattern

const count = new Map();
for (const ch of str) count.set(ch, (count.get(ch) || 0) + 1);
// plain object alt: count[ch] = (count[ch] || 0) + 1
// "are all unique?":  new Set(arr).size === arr.length

Watch out: map/filter return new arrays, but mutators return something else — chaining arr.push(x).pop() breaks because push returns a number, not the array.

Map — full method list

const m = new Map();          // or new Map([["a", 1], ["b", 2]])
m.set("a", 1)                 // add/update; RETURNS the map (chainable)
m.get("a")                    // value, or undefined if missing
m.has("a")                    // true/false
m.delete("a")                 // remove one key; returns true if it existed
m.clear()                     // remove everything
m.size                        // count (property, no parens)

// ITERATE — insertion order is preserved
for (const [k, v] of m) { ... }        // entries by default
for (const k of m.keys()) { ... }
for (const v of m.values()) { ... }
m.forEach((v, k) => { ... })           // note: value first, key second!

[...m]            // → [[k, v], ...]   array of entries
[...m.keys()]     // → [k, ...]
[...m.values()]   // → [v, ...]
Object.fromEntries(m)  // Map → plain object (string/number keys)

Map vs {}: any key type (objects, numbers stay numbers), guaranteed insertion order, real .size, and no prototype-key collisions ("toString" etc.). Prefer it for frequency counts and adjacency lists.

Set — full method list

const s = new Set();          // or new Set([1, 2, 2, 3]) → {1, 2, 3}
s.add(1)                      // add; RETURNS the set (chainable), ignores dups
s.has(1)                      // true/false — O(1) membership
s.delete(1)                   // remove; returns true if it existed
s.clear()                     // remove everything
s.size                        // count (property, no parens)

// ITERATE — insertion order
for (const v of s) { ... }
s.forEach(v => { ... })
[...s]                        // Set → array (classic dedup: [...new Set(arr)])

// common one-liners
new Set(arr).size === arr.length   // "are all elements unique?"
const inBoth = a.filter(x => setB.has(x));   // intersection of arrays

No built-in union/intersection in older runtimes — build them with has + a loop/filter as above. (Newer engines add s.union/intersection/difference, but don't rely on them in interviews.)

Snippets

Min-Heap / Priority Queue (the must-have)

class MinHeap {
  constructor() { this.h = []; }
  size() { return this.h.length; }
  peek() { return this.h[0]; }
  push(v) {
    this.h.push(v);
    let i = this.h.length - 1;
    while (i > 0) {
      const p = (i - 1) >> 1;
      if (this.h[p] <= this.h[i]) break;
      [this.h[p], this.h[i]] = [this.h[i], this.h[p]];
      i = p;
    }
  }
  pop() {
    const top = this.h[0], last = this.h.pop();
    if (this.h.length) {
      this.h[0] = last;
      let i = 0, n = this.h.length;
      while (true) {
        let l = 2*i+1, r = 2*i+2, s = i;
        if (l < n && this.h[l] < this.h[s]) s = l;
        if (r < n && this.h[r] < this.h[s]) s = r;
        if (s === i) break;
        [this.h[s], this.h[i]] = [this.h[i], this.h[s]];
        i = s;
      }
    }
    return top;
  }
}
// For a max-heap or objects, store [priority, value] and compare on [0].

Queue with O(1) dequeue (don't use array.shift in loops — it's O(n))

class Queue {
  constructor() { this.q = {}; this.front = 0; this.back = 0; }
  push(v) { this.q[this.back++] = v; }
  pop() { const v = this.q[this.front]; delete this.q[this.front++]; return v; }
  size() { return this.back - this.front; }
}

Union-Find (Disjoint Set)

class DSU {
  constructor(n) { this.p = Array.from({length: n}, (_, i) => i); this.r = new Array(n).fill(0); }
  find(x) { return this.p[x] === x ? x : (this.p[x] = this.find(this.p[x])); }
  union(a, b) {
    a = this.find(a); b = this.find(b);
    if (a === b) return false;
    if (this.r[a] < this.r[b]) [a, b] = [b, a];
    this.p[b] = a;
    if (this.r[a] === this.r[b]) this.r[a]++;
    return true;
  }
}

Trie

class TrieNode { constructor() { this.children = {}; this.end = false; } }
class Trie {
  constructor() { this.root = new TrieNode(); }
  insert(word) {
    let node = this.root;
    for (const c of word) node = node.children[c] ??= new TrieNode();
    node.end = true;
  }
  search(word) {
    let node = this.root;
    for (const c of word) { if (!node.children[c]) return false; node = node.children[c]; }
    return node.end;
  }
}

Binary search template (lower bound)

function lowerBound(arr, target) {
  let lo = 0, hi = arr.length; // hi exclusive
  while (lo < hi) {
    const mid = (lo + hi) >> 1;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid;
  }
  return lo; // first index where arr[i] >= target
}

Graph BFS / DFS skeletons

// BFS
function bfs(start, adj) {
  const seen = new Set([start]); const q = [start];
  for (let i = 0; i < q.length; i++) {
    const node = q[i];
    for (const nxt of adj[node]) if (!seen.has(nxt)) { seen.add(nxt); q.push(nxt); }
  }
}
// DFS (recursive)
function dfs(node, adj, seen = new Set()) {
  seen.add(node);
  for (const nxt of adj[node]) if (!seen.has(nxt)) dfs(nxt, adj, seen);
}

TypeScript notes for interviews

  • Most interviewers are fine with plain JS; TS types are a bonus, not required. Don't let type-wrangling eat your clock.
  • If asked to use TS, keep it light: function twoSum(nums: number[], target: number): number[].
  • const map = new Map<string, number>() reads well and shows intent.
