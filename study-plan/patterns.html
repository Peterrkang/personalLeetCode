Two Pointers Find a pair or partition in a sorted / order-able array Array, String Nested loop over all pairs — O(n²) Two pointers sweeping inward — O(n)
  1. Rely on sorted order (or sort first).
  2. Put one pointer at the left end, one at the right.
  3. Evaluate the pair against the target / condition.
  4. Need a bigger value → move left in; need smaller → move right in.
  5. Stop when the pointers cross.
  1. In sorted order, one comparison proves which side can never improve, so it's discarded.
  2. Each pointer only advances one direction, so every element is visited once → linear.
  3. No pair is ever rechecked.

Fast & Slow Pointers Detect a cycle / find the middle in O(1) space Linked List Hash set of visited nodes — O(n) space Floyd's two-speed pointers — O(1) space
  1. Advance slow by 1 and fast by 2 each step.
  2. If fast hits null, there is no cycle.
  3. If slow and fast meet, a cycle exists.
  4. To find the cycle's start, reset one pointer to head and step both by 1 until they meet.
  1. Inside a loop the fast pointer gains exactly one node per step on the slow, so it must land on it.
  2. Distances from head and from the meeting point to the cycle start are equal — that's why the reset trick works.
  3. Only two pointers are stored → constant space.

Sliding Window Best contiguous subarray / substring under a constraint Array, String Recompute every window — O(n²) One expanding/shrinking window — O(n)
  1. Advance right, adding the new element to a running state.
  2. While the window violates the constraint, advance left and undo that element's contribution.
  3. After each step, record the best valid window seen.
  1. Left only ever moves forward, so each element enters and leaves the window at most once → linear.
  2. The running state means a window is never recomputed from scratch.

Prefix Sum Many range-sum / subarray-sum queries Array, Hash Map Re-add each range on demand — O(n²) Precomputed prefixes + hash map — O(n)
  1. Build prefix[i] = sum of the first i elements.
  2. Any range sum (i, j) = prefix[j] − prefix[i].
  3. For "subarray sums to K", store counts of seen prefixes in a map and look up prefix − K.
  4. For "2d Matrix sums", use padding row/col for cleaner implementation.
  1. A range sum reuses two precomputed numbers instead of re-adding the whole range.
  2. The hash map turns "find a matching earlier prefix" into an O(1) lookup.

Binary Search Value, or smallest/largest X satisfying a monotonic predicate Array Linear scan — O(n) Halve the range each step — O(log n)
  1. Set lo/hi bounds and a predicate that goes false → true.
  2. Test mid; if it satisfies the predicate keep the left half (hi = mid), else discard it (lo = mid + 1).
  3. Converge until lo == hi — the boundary answer.
  1. Monotonicity guarantees the answer lies entirely on one side of mid, so the other half is safe to drop.
  2. Discarding half each step gives log n iterations.

Quickselect Kth smallest / largest without a full sort Array Sort then index — O(n log n) Partition, recurse one side — O(n) avg
  1. Choose a pivot (randomized to avoid worst case).
  2. Partition: smaller elements left, larger right.
  3. If the pivot's final index equals K, return it.
  4. Otherwise recurse only into the side that contains K.
  1. The half that can't contain the Kth element is never processed.
  2. Average work halves each step → O(n); a bad pivot degrades to O(n²), which randomization makes unlikely.

Merge Intervals Combine or query overlapping intervals Array Compare every interval pair — O(n²) Sort by start, single sweep — O(n log n)
  1. Sort the intervals by start.
  2. Walk through; if current.start ≤ last.end, merge by extending the end.
  3. Otherwise push the current interval as a new block.
  1. After sorting, any overlap can only be with the immediately previous interval.
  2. The sweep is O(n), so the sort dominates at O(n log n).

DFS Explore a full path / connected region Graph, Stack, Tree Re-exploring without marking — blows up Recursion/stack + visited set — O(V + E)
  1. Visit a node and mark it seen.
  2. Recurse into each unseen neighbor.
  3. Backtrack when no unseen neighbors remain.
  1. The visited set stops re-processing, bounding work to V + E.
  2. Going deep before wide maps naturally onto the call stack / an explicit stack.

Backtracking Enumerate all valid subsets / permutations / combinations Graph, Tree (decision tree) Generate everything, then filter — wasteful Choose → recurse → undo, pruning early — exponential but pruned
  1. Make a choice and append it to the current path.
  2. Recurse to extend the path further.
  3. Remove the choice (undo) before trying the next option.
  4. Prune the moment a partial path can't reach a valid answer.
  1. Early pruning skips entire invalid subtrees instead of generating them.
  2. Undoing lets a single shared path array serve every branch — no per-branch copies.

BFS Shortest path / level order in an unweighted graph or grid Graph, Queue, Tree Try all paths — exponential Queue, level by level — O(V + E)
  1. Enqueue the start and mark it seen.
  2. Pop nodes; enqueue each unseen neighbor, recording its distance / level.
  3. Finish a whole level before starting the next.
  1. The first time a node is dequeued, it was reached by a shortest unweighted path.
  2. Every node and edge is handled once → O(V + E).

Trie Prefix lookups / autocomplete over a word set Trie Scan all words per query — O(n·L) Char-by-char prefix tree — O(L) per lookup
  1. Insert each word character by character, creating nodes as needed.
  2. Flag the final node as end-of-word.
  3. Lookup / prefix queries walk down matching child nodes.
  1. Shared prefixes are stored once, so lookup cost depends only on the word's length.
  2. Prefix queries stop early at the branch point instead of scanning every word.

Union-Find Dynamic connectivity / grouping / undirected cycle detection Union-Find DFS/BFS per query — O(V + E) each Path compression + union by rank — ~O(α(n))
  1. find(x) returns the set's root, compressing the path on the way up.

  1. Path compression flattens the trees, so repeated queries become near-constant.

Heap / Top-K K largest/smallest, streaming median, merge K lists Heap Sort everything — O(n log n) Maintain a size-K heap — O(n log k)
  1. Push elements into a heap.
  2. When the size exceeds K, pop the extreme so only the best K remain.
  3. The root is the Kth element / next item to emit.
  1. You only ever order K items, never all n.
  2. log k per push beats log n whenever k ≪ n.

Dijkstra Shortest path with non-negative weights Graph, Heap Bellman-Ford relaxes all edges V times — O(VE) Greedy min-heap expansion — O(E log V)
  1. Push the source with distance 0.
  2. Pop the closest unfinalized node and finalize its distance.
  3. Relax its outgoing edges, pushing any improved neighbor distance.
  1. With non-negative edges, the closest unfinalized node's distance can never improve later, so it's safe to lock in.
  2. The min-heap returns the next closest node in log V.

Topological Sort Order respecting dependencies (DAG); detect cycles Graph Guess an order, recheck — wasteful Peel in-degree-0 nodes (Kahn's) — O(V + E)
  1. Compute in-degrees; enqueue every 0-in-degree node.
  2. Pop a node into the order; decrement each neighbor's in-degree.
  3. Enqueue any neighbor that drops to 0. Leftover nodes ⇒ a cycle.
  1. In-degree 0 means all prerequisites are already placed, so the node is safe to output.
  2. Each node and edge is touched once → O(V + E).

Monotonic Stack Next/previous greater-or-smaller, spans, histogram Stack For each element scan the rest — O(n²) One pass, monotonic stack — O(n)
  1. Iterate once, keeping the stack increasing (or decreasing).
  2. When the new element breaks the order, pop the violators — the new element is their answer.
  3. Push the new element.
  1. Each element is pushed and popped at most once → linear.
  2. The stack invariant guarantees the breaking element is exactly the resolved answer.

DP – Tabulation (Bottom-Up) Optimal/count over overlapping subproblems, iteratively Array (table) Plain recursion recomputes states — O(2ⁿ) Fill a table in dependency order — O(#states × transitions)
  1. Define the state and its base cases.
  2. Fill the table from the base cases upward to the target.
  3. Often keep only the last row / few rows to shrink space.
  1. Each state is computed exactly once in order, so nothing is recomputed.
  2. The iterative form avoids the recursion stack and enables O(1)-row space.

DP – Memoization (Top-Down) Optimal / count / reachability over overlapping subproblems Array / Map (cache) Naive recursion — O(2ⁿ) Recurse + cache each state — O(#states × work)
  1. Express the answer as a recursion on a clear state.
